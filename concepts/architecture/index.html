    Architecture¶

    This repo is organized around the measurement & attestation path:

    • runtime/ ‚Äî Œît scheduler + sliding windows
    • plant/ ‚Äî software plant (E/T/R + exchange), swappable later for hardware via a PlantAdapter
    • lmeas/ ‚Äî ‚ÄúL‚Äù measurement (two estimators), partitioning, and metrics
    • guardrails/ ‚Äî enclave-like LREG, hash-chained audit, smell-tests
    • arbiter/ ‚Äî refusal semantics + homeostasis controller
    • omega/ ‚Äî perturbations (Œ©)
    • attest/ ‚Äî device-signed indicators + exporter
    • reporting/ ‚Äî plots/tables for the paper

    Data Flow (per tick)¶

    1. scheduler ticks at fixed Œît.
    2. Controller reads state from plant.adapter, predicts risk, computes action, writes actuators.
    3. Window buffer ingests current state.
    4. When window is full, lmeas.estimators computes L_loop, L_ex ‚Üí M(dB).
    5. Smell tests applied; raw values go to LREG (write-only).
    6. Audit events appended (window measured, exports).
    7. attest.exporter periodically emits device-signed indicators (NC1 bit, SC1 bit, Mq, counter).
    8. reporting uses the audit + indicators to render figures/tables.

    All raw L stays inside the process-local LREG boundary ‚Äî exported data are derived indicators only.
